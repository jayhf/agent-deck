<objective>
Add a CI release asset validation step and harden the install script.

Purpose: Prevent empty releases from reaching users (#332). The release.yml workflow already automates GoReleaser on tag push, which solves the root cause (forgetting to run `make release-local`). This plan adds a safety net: a post-GoReleaser step that verifies all 4 expected platform tarballs (darwin_amd64, darwin_arm64, linux_amd64, linux_arm64) exist in the release, and fails the workflow if any are missing. It also improves the install script's error messaging when assets are absent.

Output: Updated release.yml with asset validation step, improved install.sh error handling.
</objective>

<task type="auto">
  <name>Task 1: Add post-release asset validation to CI workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Add a new step after the "Run GoReleaser" step in `.github/workflows/release.yml` named "Validate release assets". This step must:

1. Extract the version from the tag: `VERSION="${GITHUB_REF_NAME#v}"`
2. Use `gh api` (authenticated via GITHUB_TOKEN) to list assets on the release for tag `$GITHUB_REF_NAME`:
   ```
   gh api repos/${{ github.repository }}/releases/tags/$GITHUB_REF_NAME --jq '.assets[].name'
   ```
3. Check for the existence of exactly these 4 expected tarballs:
   - `agent-deck_${VERSION}_darwin_amd64.tar.gz`
   - `agent-deck_${VERSION}_darwin_arm64.tar.gz`
   - `agent-deck_${VERSION}_linux_amd64.tar.gz`
   - `agent-deck_${VERSION}_linux_arm64.tar.gz`
4. Also check for `checksums.txt` (produced by GoReleaser).
5. If any of the 4 platform tarballs are missing, print a clear error listing which ones are missing and exit 1.
6. If all present, print a success summary with asset count.

The step needs `GITHUB_TOKEN` in its env (already available from the workflow `permissions: contents: write`). Use `gh` CLI which is pre-installed on `ubuntu-latest` runners. Set `GH_TOKEN` env for `gh` auth:
```yaml
env:
  GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Do NOT modify any existing steps. Only append the new validation step after "Run GoReleaser".
  </action>
